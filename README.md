## System-wide philosophy
- the system is bound to change over time. When possible, decisions will be made to help the system withstand change. But it is also okay to introduce major changes, but these changes should be limited. To add some structure to the decision-making process, breaking changes are only allowed at the new year
## General note-taking philosophy

- Templates are bound to change over time. Therefore they should only contain autogenerated summaries via code cells, dataview, or other plugins. This way, all objects link back to a "quick reference", and that reference is always up to date with the latests version of the template

## Current Taxonomy

The differentiation between tags and links is difficult. Links are for specific nouns, and tags are for classes of things that are a superset of more specific nouns. For example, [[ChatGPT]] would be represented as a link but #llms would be a tag. Also, more fuzzy concepts, like #digital-infrastructure are definitely tags. Links are for pages that may ultimately turn into Concepts, while tag are for pages that will ultimately be Topics.  This distinction is intentionally vague and open to my interpretation. The decision is ultimately based on my judgement.

Is this distinction useful? I'm not sure, but I've had Obsidian a while and nothing more clever is coming to me about how to usefully separate the two. It may not become evident until I create different templates for the two types.
